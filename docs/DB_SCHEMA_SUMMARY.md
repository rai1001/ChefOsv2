# DB Schema Summary (se actualiza durante tasks)

## Tablas base

- **hotels**: `id uuid pk`, `name text`, `created_at timestamptz default now()`. RLS enabled. Policies: miembros del hotel pueden leer; `service_role` puede todas las operaciones.
- **profiles**: `id uuid pk references auth.users`, `hotel_id uuid fk hotels not null`, `role text check in (admin, manager, purchasing, warehouse, kitchen) default manager`, `created_at timestamptz default now()`. Indice `profiles_hotel_id_idx`. RLS enabled. Policies: usuario puede `select/update` su propio registro; admins del mismo hotel pueden gestionar (`all`) perfiles del hotel; `insert` solo `service_role`.
- **audit_log**: `id uuid pk default gen_random_uuid()`, `hotel_id uuid fk hotels not null`, `actor_user_id uuid references auth.users not null`, `action text`, `entity text`, `entity_id text`, `payload jsonb`, `created_at timestamptz default now()`. Indice `audit_log_hotel_id_idx`. RLS enabled. Policies: `select` solo admin/manager del mismo hotel; `insert` solo `service_role` (MVP). Se usa para auditar cambios criticos.
- **events**: `id uuid pk default gen_random_uuid()`, `hotel_id uuid fk hotels not null`, `title text`, `type text`, `start_at timestamptz`, `end_at timestamptz`, `guests_estimate int`, `status text check in (draft, confirmed, cancelled, closed) default draft`, `notes text`, `created_at timestamptz default now()`. Indices: `events_hotel_id_idx`, `events_status_idx`, `events_start_at_idx`. RLS enabled. Policies: miembros del hotel pueden leer; admin/manager del hotel pueden escribir/confirmar/cancelar. Trigger `trg_events_status_audit` (security definer) inserta en `audit_log` cuando cambia `status`.
- **production_orders**: `id uuid pk default gen_random_uuid()`, `hotel_id uuid fk hotels not null`, `event_id uuid fk events`, `title text not null`, `status text check in (planned, in_progress, done) default planned`, `notes text`, `created_at timestamptz default now()`. Indices: `production_orders_hotel_id_idx`, `production_orders_event_id_idx`, `production_orders_status_idx`. RLS enabled. Policies: miembros del hotel pueden leer; admin/manager/kitchen pueden escribir/actualizar.
- **products**: `id uuid pk default gen_random_uuid()`, `hotel_id uuid fk hotels not null`, `name text`, `category text`, `unit text check in (unidad, kg, l, pieza)`, `barcode text`, `active boolean default true`, `created_at timestamptz default now()`. Indices: `products_hotel_id_idx`, `products_name_trgm_idx` (requiere `pg_trgm`), `products_barcode_idx`. RLS enabled. Policies: miembros del hotel pueden leer; admin/manager/purchasing pueden escribir.
- **suppliers**: `id uuid pk default gen_random_uuid()`, `hotel_id uuid fk hotels not null`, `name text`, `email text`, `phone text`, `lead_time_hours int >=0`, `order_cutoff_time text`, `delivery_days jsonb`, `created_at timestamptz default now()`. Indices: `suppliers_hotel_id_idx`, `suppliers_name_trgm_idx`. RLS enabled. Policies: miembros del hotel leen; admin/manager/purchasing escriben.
- **purchase_orders**: `id uuid pk default gen_random_uuid()`, `hotel_id uuid fk hotels`, `event_id uuid fk events`, `supplier_id uuid fk suppliers`, `status text in (created, approved, sent, received_partial, received_complete, cancelled) default created`, `total_estimated numeric`, `pdf_url text`, `created_at timestamptz default now()`. Indices: hotel, supplier, status. RLS: miembros leen; admin/manager/purchasing/warehouse pueden escribir/actualizar.
- **purchase_order_items**: `id uuid pk`, `order_id uuid fk purchase_orders on delete cascade`, `product_id uuid fk products`, `qty numeric`, `unit_price_est numeric`, `notes text`. RLS enlazada al pedido y roles admin/manager/purchasing/warehouse.
- **inventory_lots**: `id uuid pk default gen_random_uuid()`, `hotel_id uuid fk hotels`, `product_id uuid fk products`, `qty numeric >=0`, `unit text (unidad/kg/l/pieza)`, `expires_at timestamptz`, `received_at timestamptz default now()`, `source_order_id uuid fk purchase_orders`, `receipt_url text`, `created_at timestamptz default now()`. RLS: miembros leen; admin/manager/warehouse escriben.
- **inventory_movements**: `id uuid pk default gen_random_uuid()`, `hotel_id uuid fk hotels`, `product_id uuid fk products`, `lot_id uuid fk inventory_lots nullable`, `type text in (in,out,adjust,waste)`, `qty numeric`, `reason text`, `actor_user_id uuid fk auth.users`, `created_at timestamptz default now()`. RLS: miembros leen; insert admin/manager/warehouse/kitchen; update admin/manager/warehouse.

## Notas
- Multi-tenant: todas las tablas de negocio incluyen `hotel_id` y se filtran por RLS.
- Roles validos: admin, manager, purchasing, warehouse, kitchen.
- PDF de pedidos: Edge Function `purchase-order-pdf` genera un placeholder y guarda en bucket Storage `purchase-order-pdfs` con URL firmada.
