# GAPS PRD vs Implementación — auditoría 2026-04-16

> Cruce del PRD (1.312 líneas) contra el código actual (migraciones 00001-00038 + UI). Estado **antes** de la demo Iago (sábado 19 abr).

## TL;DR

13/16 módulos al >80%. Lo crítico para demo está OK. **Para operación real post-piloto** faltan 4 features estrella:

1. **OCR + matching automático albaranes** (M4)
2. **Conciliación tres-vías** PO ↔ GR ↔ Factura (M4)
3. **Importación masiva** Excel/CSV/PDF (M3 + M4)
4. **IA real en agentes + ML forecasting** (M11 + M15) — diferido a Q3 2026

## Sección A — Features estrella faltantes

### 1. OCR de albaranes (M4 — línea 416-437 del PRD)

**Lo que pide el PRD:**
- Foto del albarán → extracción automática (proveedor, productos, cantidades, precios, fecha)
- Matching contra `product_aliases` con confidence score
- Revisión manual de items no reconocidos (drag & drop)
- Auto-actualización de precios vs `supplier_offers`
- Creación automática del `goods_receipt`
- Trigger: precio cambió → recalcular `recipe_ingredients.unit_cost` → recalcular escandallos

**Lo que hay hoy:**
- Tabla `goods_receipts` con campo `ocr_data jsonb` (vacío, no se usa)
- Cero RPC, cero Edge Function, cero UI

**Lo que falta:**
- Edge Function `ocr-receipt` con Claude Vision o Google Document AI
- RPC `process_ocr_receipt(hotel_id, order_id, ocr_data)` que:
  - Crea `goods_receipt` + `goods_receipt_lines` desde JSON extraído
  - Detecta cambios de precio vs `purchase_order_lines.unit_price`
  - Llama a `update_recipe_costs_from_price_change(product_id, new_price)` en cascada
- RPC `match_product_by_alias(alias_text)` con confidence score (Levenshtein o embedding)
- Botón "📷 Subir albarán" en `/procurement/orders/[id]`
- Pantalla revisión OCR con drag & drop de items no reconocidos
- Trigger PostgreSQL: cambio en `recipe_ingredients.unit_cost` → recalcula `recipes.cost_per_serving`

**Esfuerzo:** ~8-12h (depende de la API OCR elegida)

---

### 2. Conciliación tres-vías PO ↔ GR ↔ Factura (M4 — línea 1062 del PRD)

**Lo que pide el PRD:**
- Pedido emitido (PO) ↔ Albarán recibido (GR) ↔ Factura del proveedor (Invoice)
- Detectar diferencias en cantidades, precios, descuentos
- Disparar acciones: alerta, retención de pago, reclamo a proveedor, ajuste contable

**Lo que hay hoy:**
- `purchase_orders` y `goods_receipt_lines` ✓
- **Tabla `purchase_invoices` NO existe**
- Cero RPC, cero UI

**Lo que falta:**
- Migración nueva: tabla `purchase_invoices` (invoice_number, date, total, payment_terms) + `purchase_invoice_lines` (qty, unit_price, line_total)
- RPC `reconcile_three_way(order_id)` que devuelva por línea:
  - Cantidad pedida vs recibida vs facturada
  - Precio pedido vs cobrado en factura
  - Variance % y flag (ok / minor / major)
- RPC `get_reconciliation_exceptions(hotel_id)` para dashboard
- UI `/procurement/reconciliation` con tabla side-by-side
- Auto-generar `alerts` cuando variance > 5%
- Workflow de aprobación de factura para pago

**Esfuerzo:** ~12-16h

### 3. Importación masiva Excel/CSV/PDF (M3 + M4)

**Lo que pide el PRD (líneas 660, 421):**
- Carga de catálogo de productos completo desde Excel del proveedor
- Carga de tarifas de precios desde PDF/Excel
- Validación de duplicados, normalización de unidades, matching alias
- Carga histórica de albaranes desde escaneo masivo

**Lo que hay hoy:**
- RPC `import_products_bulk(hotel_id, products jsonb)` parcial (sí existe en 00007)
- Cero UI para subir Excel
- Cero parser CSV/PDF
- Cero validación duplicados con UI

**Lo que falta:**
- Pantalla `/catalog/import` con dropzone + preview tabla
- Edge Function `parse-excel` (SheetJS) y `parse-pdf` (pdf-parse)
- RPC `import_supplier_offers_bulk(hotel_id, supplier_id, offers jsonb)`
- UI de revisión de duplicados/conflictos antes de commit
- Importación de tarifas con detección automática de columnas (proveedor envía Excels distintos)

**Esfuerzo:** ~10-14h
